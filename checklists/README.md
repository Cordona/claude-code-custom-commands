# Command Quality Checklists

*Simple, focused validation for professional custom command development*

---

## 🎯 **3 Essential Checklists**

### **📋 [Pre-Development Checklist](development/01-pre-development.md)**
*Planning, requirements validation, and team alignment before implementation*

**Use this checklist to:**
- Define clear requirements and success criteria
- Plan command architecture and security approach
- Align with team standards and workflows
- Ensure readiness before starting development

**When to use:** Before writing any command code

---

### **✅ [Development Checklist](development/02-development.md)**
*Comprehensive daily validation for professional command development*

**🌟 The Daily Genius Checklist** - This is your primary quality tool!

**Use this checklist for:**
- Core command quality and design validation
- Built-in security practices and data protection
- Performance optimization and efficiency
- Implementation standards and team integration
- Quality validation and testing

**When to use:** During command development (daily validation)

---

### **🚀 [Post-Development Checklist](development/03-post-development.md)**
*Testing, deployment validation, and monitoring setup after implementation*

**Use this checklist to:**
- Comprehensive testing and validation
- Security and risk verification
- Deployment readiness confirmation
- Monitoring and maintenance setup

**When to use:** Before deploying or sharing commands

---

## 🎯 **How to Use These Checklists**

### **For Individual Developers**
1. **Start with Pre-Development** - Plan before coding
2. **Use Development Daily** - Your primary quality validation tool
3. **Complete Post-Development** - Ensure deployment readiness

### **For Code Reviews**
- **Use Development Checklist** as your review guide
- **Verify Post-Development** items before approval
- **Ensure Pre-Development** planning was thorough

### **For Teams**
- **Pre-Development** - Team alignment and standards compliance
- **Development** - Consistent quality across all team members
- **Post-Development** - Team deployment and maintenance readiness

## 💡 **Design Philosophy**

### **Simple & Focused**
- **3 checklists total** - no decision paralysis
- **Process-based organization** - follows natural development flow
- **No duplication** - each checklist covers unique phase

### **Comprehensive Coverage**
- **Security built-in** - not separated from development process
- **Performance integrated** - optimization during development
- **Team standards included** - collaboration part of quality

### **Practical Daily Use**
- **Development Checklist** serves as daily validation
- **Quick reference** without overwhelming detail
- **Links to guides** for detailed information when needed
